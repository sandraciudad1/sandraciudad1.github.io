Descripción del proyecto


Este proyecto consistió en la implementación de una versión digital del clásico Whack-a-Mole. El objetivo fue diseñar un juego en 2D con mecánicas basadas en la aparición aleatoria de objetivos que deben ser golpeados por el jugador en un tiempo limitado, aplicando principios fundamentales de programación, gestión de eventos e interacción con objetos en pantalla.

El sistema principal se estructuró mediante el uso de prefabs reutilizables y lógica de posicionamiento aleatorio, utilizando la clase Random.Range() y una colección de Vector3 para definir las posibles ubicaciones de aparición del topo. El control de interacción se gestionó con OnMouseDown() sobre un Collider 3D, lo que permitió detectar clics en los objetos dentro de una escena 2D con precisión y bajo coste computacional. Se estableció una cuenta atrás general del juego, controlada por un temporizador global, y una cuenta atrás interna por cada aparición del topo, que se reinicia tras cada golpe o tras el paso del tiempo sin interacción.

La interfaz de usuario fue diseñada con Canvas y TextMeshPro para mostrar el tiempo restante y la puntuación del jugador en tiempo real. Además, se implementó una lógica de finalización que desactiva la aparición de topos cuando finaliza el temporizador global. El proyecto se desarrolló siguiendo principios de programación orientada a objetos en C#, priorizando la modularidad, claridad del código y facilidad para futuras ampliaciones, como la aparición de múltiples topos, incremento de dificultad progresiva o incorporación de power-ups.
