Descripción del proyecto


Este proyecto corresponde al desarrollo de un videojuego del género Shoot’em Up, diseñado en Unity con un enfoque en la eficiencia y la arquitectura escalable. El jugador controla un cañón terrestre cuyo objetivo es eliminar enemigos representados por rocas de distintos tamaños que se desplazan por la pantalla. El sistema de disparo incluye dos modos: un disparo primario, que consiste en ráfagas continuas de balas individuales, y un disparo secundario de uso limitado, que permite limpiar la pantalla de enemigos simultáneamente.

El diseño del sistema de disparo se implementó con lógica de control basada en Input.GetKey para ráfagas continuas y Input.GetKeyDown para activaciones puntuales. La eficiencia del juego se garantizó mediante el uso de piscinas de objetos tanto para balas como para enemigos, minimizando la creación y destrucción de GameObjects en tiempo de ejecución y optimizando el rendimiento en dispositivos con recursos limitados.

Se incorporó el patrón Singleton para centralizar la gestión de componentes clave como el GameManager, el BombManager y los controladores de generación de enemigos y proyectiles. Además, se utilizaron delegados para implementar un sistema de notificación reactivo que permite a los distintos tipos de enemigos responder de forma diferenciada ante el uso del disparo secundario, manteniendo el bajo acoplamiento entre clases.

Este proyecto sirvió como base para experimentar con principios de diseño como el principio de responsabilidad única y la segregación de interfaces, con vistas a su portabilidad a dispositivos móviles en fases posteriores.
