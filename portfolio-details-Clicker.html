Descripción del proyecto


Este proyecto consistió en el desarrollo de un videojuego tipo clicker en Unity, centrado en la gestión de una cafetería. El jugador asume el rol de administrador del local y debe atender pedidos de clientes mediante interacciones rápidas en pantalla, acumulando beneficios que permiten invertir en mejoras progresivas. El objetivo es optimizar la eficiencia del negocio, incrementar la satisfacción del cliente y desbloquear nuevas funcionalidades conforme se avanza.

La lógica central se construyó mediante programación orientada a objetos en C#, integrando un sistema de progresión que permite mejorar máquinas de café, ampliar la variedad de toppings y desbloquear una segunda tienda especializada en donuts, manteniendo la misma mecánica de gestión. La comunicación entre escenas se gestionó mediante objetos persistentes y estructuras de datos compartidas para mantener el estado del juego entre el menú, el gameplay y el inventario.

El proyecto incluye tres escenas principales: una escena de menú con acceso al juego y a los récords guardados, una escena de gameplay donde se desarrolla la actividad principal mediante eventos de input, y una escena de inventario que se desbloquea tras alcanzar ciertos hitos, permitiendo al jugador seleccionar y aplicar mejoras activas. Todo el sistema se acompaña de una interfaz de usuario interactiva con Canvas y TextMeshPro, así como una lógica de control de fases y economía interna.

El juego fue diseñado para ser escalable y modular, permitiendo futuras ampliaciones en la estructura de tiendas, tipos de cliente, mecánicas de atención y elementos decorativos, todo ello bajo buenas prácticas de programación y control del flujo de datos entre escenas.
